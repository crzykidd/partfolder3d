## A. Security

Deployment context for severity calibration: self-hosted, single household, behind the owner's
HTTPS reverse proxy; Postgres and Redis are **not** port-published (internal Docker network
only). Overall residual risk is **low-to-moderate for that context** — but several items become
the dominant risk the moment the instance is exposed to any less-trusted importer/user.

### Fix promptly

- [x] **[high] SSRF: scraped image URLs fetched at commit with no guard, no redirect re-check, no size cap.**
  `backend/app/routers/import_sessions/sessions.py:695-711` does
  `httpx.Client(...).get(si.path, follow_redirects=True)` then `img_dest.write_bytes(r.content)`
  with **no `assert_safe_url()`** (unlike session-*create*, which guards at `sessions.py:134-140`).
  Image URLs come from the target page's `og:image`/`<img>` (attacker-controlled if the imported
  page is malicious) and from AgentQL. → blind SSRF to `169.254.169.254`/RFC1918/internal hosts;
  `r.content` fully buffered with no cap → worker RAM/disk DoS; redirects can 30x into the internal
  range. **Fix:** call `assert_safe_url(si.path)` before fetch; disable redirects (or re-guard each
  hop); stream with a byte cap (`SCRAPE_MAX_IMAGE_MB`) via `client.stream()`; enforce `Content-Type: image/*`.

- [x] **[high] SSRF guard is pre-flight-only → bypassable via redirects and DNS-rebinding (TOCTOU).**
  `scraper.py:246-255` (`scrape_url`) and `scraper.py:87` (`_get_robots`) call `assert_safe_url()`
  which resolves+checks DNS, then open a *separate* `httpx.Client(follow_redirects=True)` that
  re-resolves and follows redirects **with no re-validation**. Defeated by (a) a public host that
  `302`s to `http://10.0.0.5/…`, or (b) DNS rebinding (public A for the pre-check, private A on
  connect). The module docstring itself acknowledges it's "purely DNS-based pre-flight." **Fix:**
  `follow_redirects=False` + manually re-`assert_safe_url` each `Location`; or pin the connection to
  the vetted IP via a custom transport/resolver. Apply to `_get_robots` too.

- [x] **[med] `javascript:`-scheme XSS via `source_url` / `profile_url` rendered as anchor `href`.**
  Rendered directly into `href` with no scheme allow-listing, incl. the **unauthenticated** public
  share page: `frontend/src/pages/PublicSharePage.tsx:521,587`, `item/ItemMetadata.tsx:117`,
  `CreatorPage.tsx:161`, plus `ImportsPage.tsx:238`, `TitleStep.tsx:264`, `SummaryStep.tsx:151`.
  Backend sets the field with no validation (`items.py:752-753`) and scrapers populate it. React
  does not block `javascript:` hrefs; the CSRF cookie (`pf3d_csrf`) is deliberately JS-readable, so
  one click = full authenticated request forgery. **Fix:** reject non-http/https schemes on the
  backend where these fields are set (item PATCH, scraper ingestion, creator create) **and** add a
  frontend `safeHref()` helper used on every external `href`.

### Harden (defense-in-depth; lower urgency for the trusted-user model)

- [x] **[med] Redis has no `requirepass` + arq deserializes jobs with pickle → latent worker RCE.**
  `docker-compose.yml:46-56` (no `--requirepass`), arq defaults to `pickle.loads` for job bodies
  (never overridden). Any process able to write the Redis queue key → arbitrary code execution in
  the worker (which can write the whole library + `secret.key`). Contained *today* only because
  Redis isn't network-reachable. **Fix:** set a Redis password in `REDIS_URL`; switch arq to a JSON
  serializer (all job args here are ints/strings).

- [x] **[med] nginx sends no security headers.** `nginx/nginx.conf` has no `X-Frame-Options`/
  `frame-ancestors`, `Content-Security-Policy`, `X-Content-Type-Options: nosniff`, or
  `Referrer-Policy`. → clickjacking of the authed SPA; MIME-sniffing. **Fix:** add
  `X-Frame-Options DENY`, `X-Content-Type-Options nosniff`, `Referrer-Policy no-referrer`, and a
  conservative CSP at the `server` level (`always`). Don't assume the outer proxy adds them.

- [x] **[med] Weak default DB password propagates.** `POSTGRES_PASSWORD:-changeme` in both compose
  files (`docker-compose.yml:38,70,113`; dev `:31,67`) and baked into `DATABASE_URL`
  (`config.py:22`). `.env` is `required: false`, so skipping it runs Postgres with `changeme`.
  Contained to the Docker network today. *(Also flagged by the backend agent — same root cause.)*
  **Fix:** fail fast at startup if `POSTGRES_PASSWORD` is unset/`changeme` in a non-dev profile, or
  generate one; document as required.

- [x] **[med] Raw exception text leaked into HTTP error details; no global exception handler.**
  ~13 sites interpolate `{exc}` into `detail=`, and `main.py` has no catch-all handler. Verified:
  `jobs.py:315-319,428`, `import_sessions/sessions.py:854-857`, `items.py:740,844,1523`,
  `issues.py:260`, `scheduled_jobs.py:104`, `backup.py:140`, `import_sessions/__init__.py:61,152`,
  `sessions.py:142`. **Security-adjacent:** `ssrf_guard.py:30` raises `HTTPException(400, str(exc))`,
  leaking internal-network block reasons to import users. **Fix:** add a global handler returning a
  generic 500 + `log.exception`; replace `{exc}` with generic messages; scrub the SSRF-block message.
  *(Cross-listed in §E.3.)*

- [x] **[med] CI/CD supply chain: unpinned actions + `packages: write` on every dev push.**
  All workflows use mutable major tags (`actions/checkout@v5`, `docker/build-push-action@v7`, …);
  `publish.yml:8-9` triggers on `push:[dev,main]` with `permissions: packages: write`, so any dev
  push builds+pushes `:dev`/`sha-` images; `ci.yml`/`dev-checks.yml` declare no top-level
  `permissions:`. (Mitigated: no `pull_request_target`; prod consumes `:latest` = main/release only,
  so dev can't poison prod.) **Fix:** SHA-pin third-party actions (Dependabot can maintain); add
  `permissions: contents: read` to `ci.yml`/`dev-checks.yml`.

- [x] **[med] No authorization model on the core catalog; API keys are unscoped.** Every mutation
  (`items.py` create/update/delete/upload/rename/image; `shares.py:576` mint *public* share for
  *any* item) gates only on `get_current_user` — no ownership/role check; a Bearer API key resolves
  to the full user (`auth/deps.py:63-80`), no per-key scope. Acceptable for "few trusted users" but
  **undocumented**. **Action (decide + document):** record the "all authenticated users are fully
  trusted; API keys are full-power" assumption in `docs/decisions.md`; only build ownership/role
  gates + a `scope` column if the instance will ever host less-trusted users.

- [x] **[med] Redis pool leak: 32 `create_pool(...)` sites, zero `try/finally`.**
  e.g. `jobs.py:303-306` — if `enqueue_job()` raises, `aclose()` never runs. Same in `reviews.py`,
  `backup.py`, `shares.py`, `scheduled_jobs.py`, `downloads.py`, `import_sessions/helpers.py`. Also
  wasteful (fresh pool per request). **Fix:** one arq pool at app-lifespan startup, injected; or
  minimally wrap each in `try/finally`. *(Cross-listed in §E.3.)*

### Lower severity

- [x] **[low] Cross-user tampering with print records.** `print_records.py` update (261)/delete (304)
  resolve via `_get_record_or_404(record_id, item.id, ...)` with **no `logged_by_id` check** → any
  user can edit/delete another user's record; list/get return `visibility="private"` records to any
  authenticated user (public share path correctly filters, `shares.py:292`). **Fix:** enforce
  `logged_by_id == user.id or admin` on update/delete.
- [x] **[low] First-run setup TOCTOU can mint a second admin.** `setup.py:80→93-100` count-then-insert
  with no lock; two concurrent `POST /api/setup` both pass `count()==0`. Narrow (first-run window
  only). **Fix:** unique partial index / `INSERT … WHERE NOT EXISTS` / advisory lock → second req 409s.
- [ ] **[low] No rate limiting on login or AI endpoints; login user-enumeration timing.**
  `auth.py:62` no throttle; `auth/provider.py:43-46` skips argon2 verify on unknown email (timing
  oracle); `ai_actions.py` suggest-tags/cleanup/summarize have no per-user spend cap. **Fix:** per-IP/
  account login throttle; dummy `verify_password` on unknown email; per-user AI budget.
- [x] **[low] Password reset/change doesn't invalidate existing sessions.** `password_reset.py:126`
  sets the hash but never deactivates `user_sessions`. **Fix:** mark all the user's `UserSession`
  rows `is_active=False` on password change/reset.
- [x] **[low] Unbounded response body in `scrape_url`.** `scraper.py:255-272` reads `resp.text` with
  no size cap; non-HTML rejected only *after* buffering. → worker RAM exhaustion (OOM-kills worker).
  **Fix:** `client.stream()` + abort after N bytes; check `Content-Length`/`Content-Type` first.
- [ ] **[low] Zip caps trust declared central-directory sizes.** `archive.py:241-253,299` compute the
  bomb ratio and per-entry cap from declared `file_size`/`compress_size`; extraction
  (`:310-311 copyfileobj`) streams real bytes with no runtime counter. A hand-crafted ZIP that
  under-declares sizes bypasses the caps. **Fix:** running byte budget inside the copy loop.
- [ ] **[low] Untrusted 3MF XML parsed with lxml without hardened entity settings.**
  `threemf.py:134,298,335,424` `etree.fromstring(data)` on user `.3mf`. lxml defaults mitigate
  classic XXE but resolution isn't explicitly disabled. **Fix:** explicit
  `XMLParser(resolve_entities=False, no_network=True, load_dtd=False, dtd_validation=False)`,
  `huge_tree=False`.
- [ ] **[low] Backup archive bundles the encryption key with the encrypted secrets.**
  `worker/backup.py:118-163` exports api_keys/ai_providers (Fernet-encrypted) + users/sessions/tokens
  **and** copies `config/secret.key` into the same tarball → one leaked backup = full secret
  disclosure. (Download is `require_admin` — good.) **Fix:** document sensitivity; restrict
  `/data/backups` perms; consider a passphrase-encrypted archive or excluding `secret.key`.
- [x] **[low] Raw user URLs logged unsanitized (log injection).** `scraper.py:235,242,276,280,331`
  log URLs with `%s` un-sanitized, unlike `ssrf_guard.py:138`/`bulk_commit` which already scrub CR/LF.
  **Fix:** apply the same CR/LF-escape helper in `scraper.py`, `agentql_client.py`,
  `tasks/import_session.py`.
- [x] **[low] CORS validator permits `*` with `allow_credentials=True`.** `main.py:84-90` +
  `config.py:42-56` don't reject `"*"`; an operator setting `ALLOWED_ORIGINS=*` gets a broken config
  rather than a hard error. **Fix:** reject `"*"` when credentials are enabled.
- [ ] **[low] Mutable image tags; `X-Forwarded-Proto` fixed to `http`.** Prod compose pins images by
  tag not digest (`:latest`, `postgres:16-alpine`, …); `nginx.conf:33` sets
  `X-Forwarded-Proto $scheme`(=http) behind the outer TLS proxy. **Fix:** pin by digest; confirm the
  backend derives HTTPS from the outer proxy (or set the header explicitly).

### Solid defenses observed (do not regress these)

Path-traversal containment (`resolve()`+`is_relative_to()`, symlink-resolving) applied consistently
across downloads/shares/uploads · zip extraction hardened (zip-slip/absolute/drive-letter rejection,
file-count/uncompressed/per-entry caps, 200× bomb ratio, temp-dir→atomic move) · all served files
forced to `application/octet-stream`/`zip` (no inline render; SVG excluded from image uploads → no
stored-XSS via served content) · SQL safe (2 `text()` uses, both bound/whitelisted; no f-string SQL)
· YAML `safe_load`/`safe_dump` everywhere · crypto: argon2id + all tokens `secrets.*` 256-bit, API
keys SHA-256 (shown once), invite/reset hashed · CSRF double-submit with `hmac.compare_digest`,
httpOnly+SameSite=lax session cookie · public shares check existence+expiry+revocation every request,
strip private/fs data · render runs in a `spawn`ed subprocess with wall-clock kill + size/triangle/
thread caps · containers non-root (PUID/PGID) with hard cpu/mem limits · Postgres/Redis not
port-published · arq enqueue from API constrained to known task names + admin-gated · `ssrf_guard.py`
itself is thorough (loopback/RFC1918/link-local/IMDS/CGNAT/ULA/v4-mapped-v6/multicast/TEST-NET,
fail-closed) · no `dangerouslySetInnerHTML` anywhere · datetime fully tz-aware.
