# Task: Item file management + extraction visibility (issues #18, #19)

Add upload/delete/rename endpoints for item model files (mirroring the existing
image management API), fix extraction job invisibility, and make the item page
auto-refresh when background extraction finishes.

## Before you start

- Read `docs/atomic-moves.md` for rename conventions (stay within item dir, no path traversal).
- Read `docs/sidecar-schema.md` for sidecar sync expectations.
- Match the image upload/delete pattern in `backend/app/routers/items.py`.
- Frontend stack: Tailwind + Aurora CSS vars + TanStack Query; NO Mantine/toast.

## Working tree check

`git status --porcelain` — clean on entry.

## What to do

### A — Backend: file management endpoints

1. `POST /api/items/{key}/files` — upload file; save to item dir root; hash; create File row; enqueue analyze + render (3MF skips render via model_extensions guard); sync sidecar.
2. `DELETE /api/items/{key}/files/{file_id}` — remove file from disk (best-effort) + DB row; sync sidecar.
3. `PATCH /api/items/{key}/files/{file_id}` — rename file (basename only, keep parent dir); guard traversal + collisions; update files.path + role; sync sidecar.
4. `GET /api/items/{key}/jobs` — return active (queued/running) Job rows for the item; no admin required (just auth); used by frontend for polling.

### B — Backend: extract_archives Job tracking

- In `backend/app/worker/tasks/archive.py`, create a Job row at task start (`type="extract_archives"`) using `job_tracker.create_job/finish_job`.
- Mark succeeded at end; mark failed on exception or if item not found; mark succeeded with note if no ZIPs.
- Use `ctx.get("job_id")` for `arq_job_id`.

### C — Frontend: file management UI

- Add `uploadItemFile`, `deleteItemFile`, `renameItemFile`, `listItemJobs` to `frontend/src/lib/api/items.ts`.
- Update `DownloadsPanel.tsx`: add delete (Trash2 icon + confirm), rename (Pencil icon + inline input) to each FileRow, and upload control at bottom of the file panel.
- Update `ItemPage.tsx`: wire all three mutations; add TanStack Query poll for active item jobs (every 3 s while jobs exist); invalidate item query when jobs clear → file list auto-updates.

### D — Tests

- `backend/tests/test_item_files.py`: API tests for upload, delete, rename endpoints + extract_archives Job row test.
