# Task: Fix AI issues #16 and #17 — unsaved description and sync event-loop block

Implements two related bug fixes that both touch the AI code path. Issue #16: the
wizard's "Clean up (AI)" and "Summarize" buttons ignore text the user has typed but
not yet saved. Issue #17: `_dispatch` is synchronous and blocks the entire event loop
(including /health) when the AI provider is slow.

## What was done

### Issue #17 — async offload + timeout

- `backend/app/ai/client.py`: Added `timeout: float` parameter (default 60 s) to
  `_call_anthropic_real`, `_call_openai_real`, and `_dispatch`. The timeout is passed
  to the SDK's per-request call (`timeout=timeout`). Injectable/test callers do NOT
  receive the timeout argument so existing test lambdas need no changes.
- `backend/app/routers/ai_providers.py`: Wrapped `_dispatch` in
  `await asyncio.to_thread(...)` with `timeout=10.0` (aggressive for connectivity test).
- `backend/app/routers/ai_actions.py`: Wrapped all three public AI calls
  (`suggest_tags`, `cleanup_description`, `summarize_scrape`) in
  `await asyncio.to_thread(...)`.

### Issue #16 — pass current description/title in body

- `backend/app/routers/ai_actions.py`: Added `AiDescriptionRequest` body model
  (all fields optional). `ai_cleanup_description` and `ai_summarize_scrape` now
  accept an optional body; `body.description` and `body.title` (when non-None)
  take precedence over the persisted session values.
- `frontend/src/lib/api/ai.ts`: Added `AiDescriptionRequest` interface; updated
  `aiCleanupDescription` and `aiSummarize` to accept and send a body.
- `frontend/src/pages/import-wizard/TitleStep.tsx`: Updated `cleanupMutation` and
  `summarizeMutation` to pass `{ description, title }` from local component state.

### Tests

- `backend/tests/test_phase8_ai.py`: Added tests proving body description overrides
  the persisted value for both endpoints, and that a session with no persisted
  description (the exact bug scenario) works when the body provides one.
